\chapter{\ifenglish{Background Knowledge and Theory}\else{ทฤษฎีที่เกี่ยวข้อง}\fi}

การดำเนินโครงงานวิจัยนี้อาศัยพื้นฐานความรู้ด้านตัวแบบภาษาขนาดใหญ่ (LLMs) ความมั่นคงปลอดภัยของข้อมูล และบริบททางภาษาศาสตร์ของภาษาไทย โดยมีรายละเอียดทฤษฎีที่เกี่ยวข้องดังนี้:

\section{ตัวแบบภาษาขนาดใหญ่ (Large Language Models: LLMs)}
ตัวแบบภาษาขนาดใหญ่เป็นนวัตกรรมด้านการประมวลผลภาษาธรรมชาติ (NLP) ที่ใช้โครงข่ายประสาทเทียมขนาดมหาศาลเพื่อทำความเข้าใจและสร้างภาษา  โดยมีองค์ประกอบสำคัญดังนี้
\begin{itemize}
  \item สถาปัตยกรรม (Architecture): โมเดลส่วนใหญ่พัฒนาบนสถาปัตยกรรม Transformer ซึ่งใช้กลไก Self-attention ในการประมวลผลความสัมพันธ์ของคำในบริบทที่ยาวได้อย่างมีประสิทธิภาพ
  \item กระบวนการฝึกฝน (Training Stages): ประกอบด้วย 3 ขั้นตอนหลัก ได้แก่:
    \begin{itemize}
      \item Pre-training: การเรียนรู้จากชุดข้อมูลดิบจำนวนมหาศาลเพื่อให้โมเดลเข้าใจโครงสร้างภาษา
      \item Supervised Fine-tuning (SFT): การปรับจูนโมเดลด้วยชุดข้อมูลที่ระบุผลลัพธ์ที่ถูกต้องเพื่อให้โมเดลตอบสนองต่อคำสั่งได้ตรงประเด็น
      \item RLHF (Reinforcement Learning from Human Feedback): การปรับแต่งผลลัพธ์สุดท้ายด้วยคำแนะนำจากมนุษย์ เพื่อให้โมเดลมีความปลอดภัยและเป็นประโยชน์สูงสุด
    \end{itemize}
  \item ตัวแบบภาษาที่ใช้ในการศึกษา: โครงงานนี้มุ่งเน้นศึกษาโมเดลขนาดเล็กที่ประหยัดทรัพยากรแต่มีประสิทธิภาพสูง ได้แก่:
    \begin{itemize}
      \item \texttt{llama-3.1-8b-instant}: โมเดลขนาด 8 พันล้านพารามิเตอร์ที่มีความเร็วและประสิทธิภาพสูง % chktex 8
      \item \texttt{meta-llama-llama-4-maverick-17b-128e-instruct}: โมเดลขนาดกลางที่มีความแม่นยำและการจัดลำดับเหตุผลที่ซับซ้อน % chktex 8
      \item \texttt{moonshotai-kimi-k2-instruct-0905}: โมเดลที่มีจุดเด่นในการประมวลผลบริบทที่ยาวและรองรับหลายภาษา % chktex 8
    \end{itemize}
\end{itemize}
\section{ความปลอดภัยและความเป็นส่วนตัวของข้อมูลใน LLMs}
ความเป็นส่วนตัวของข้อมูลใน LLMs เกี่ยวข้องกับการป้องกันการรั่วไหลของข้อมูลที่อ่อนไหว (Sensitive Information)  ซึ่งมักเกิดจากรูปแบบการโจมตีดังนี้:

\begin{itemize}
  \item Data Privacy Leakage: การที่โมเดลจดจำข้อมูลส่วนบุคคล (PII) จากชุดข้อมูลฝึกฝนและแสดงผลลัพธ์ออกมาโดยไม่ตั้งใจ
  \item Data Extraction Attack (DEA): การออกแบบคำสั่งเพื่อล่อลวงให้โมเดลคายข้อมูลลับ เช่น อีเมล หรือข้อมูลติดต่อส่วนตัว
  \item Jailbreak Attack (JA): การใช้เทคนิคการป้อนคำสั่ง (Prompt Injection) เพื่อหลีกเลี่ยงกฎเกณฑ์ด้านความปลอดภัย (Safety Guardrails) ทำให้โมเดลตอบสนองต่อคำสั่งที่ควรถูกระงับ
\end{itemize}

\section{บริบทภาษาไทยและระดับภาษา (Thai Context and Linguistic Registers)}
ความซับซ้อนของภาษาไทยเป็นปัจจัยสำคัญที่ทำให้มาตรฐานการประเมินภาษาอังกฤษไม่สามารถนำมาใช้ได้อย่างครอบคลุม:
\begin{itemize}
  \item ความสำคัญของบริบทไทย: ภาษาไทยมีความกำกวมและการใช้คำที่ขึ้นอยู่กับบริบททางวัฒนธรรมสูง ทำให้การตรวจจับภัยคุกคามทำได้ยากกว่าภาษาที่มีโครงสร้างตายตัว
  \item ระดับภาษา 5 ระดับ (Linguistic Registers): โครงงานนี้แบ่งการทดสอบออกเป็น 5 ระดับเพื่อประเมินความเปราะบางอย่างละเอียด:
    \begin{enumerate}
      \item ภาษาระดับพิธีการ: ใช้ในงานพระราชพิธีหรือโอกาสสำคัญยิ่ง
      \item ภาษาระดับทางการ: ใช้ในหนังสือราชการหรือบทความวิชาการ
      \item ภาษาระดับกึ่งทางการ: ใช้ในการประชุมหรือการสื่อสารที่เป็นงานเป็นการ
      \item ภาษาระดับไม่เป็นทางการ: ใช้ในการสนทนาทั่วไปหรือการทำงาน
      \item ภาษาระดับกันเอง: ใช้ในครอบครัวหรือกลุ่มเพื่อนสนิท ซึ่งมักมีคำแสลงหรือภาษาปาก
    \end{enumerate}
\end{itemize}

\section{กลไกการป้องกัน (Defense Mechanisms)}
กลไกป้องกันที่นำมาศึกษาเพื่อเพิ่มความปลอดภัยให้กับระบบมี 2 รูปแบบหลัก:

\begin{itemize}
  \item Input Scrubbing: การใช้เทคนิค Regex (Regular Expression) และ NER (Named Entity Recognition) เพื่อตรวจจับและปกปิดข้อมูลส่วนบุคคลก่อนที่คำสั่งจะถูกส่งไปยังโมเดล
  \item Defensive Prompting: การกำหนดคำสั่งในระดับระบบ (System Prompt) เพื่อย้ำเตือนและควบคุมพฤติกรรมของโมเดลให้ปฏิเสธการให้ข้อมูลที่อ่อนไหวหรือคำสั่งที่ผิดกฎความปลอดภัย
\end{itemize}
\section{งานวิจัยที่เกี่ยวข้อง (Related Works)}
งานวิจัยนี้ได้ศึกษาและอ้างอิงแนวทางจากวรรณกรรมที่สำคัญ ดังนี้:
\begin{itemize}
  \item LLM-PBE~\cite{gu2024llmpbe}: งานวิจัยต้นแบบที่นำเสนอกรอบการประเมินความเป็นส่วนตัวข้อมูลใน LLMs ซึ่งเป็นฐานสำคัญในการพัฒนาชุดทดสอบภาษาไทยในโครงงานนี้
  \item สรุปงานวิจัยอื่น ๆ: จากการศึกษาเพิ่มเติมพบว่าภัยคุกคามใน LLMs มีวิวัฒนาการอย่างต่อเนื่อง เช่น งานวิจัย PreCurious ที่ศึกษาเรื่องกับดักความเป็นส่วนตัวในโมเดลที่ผ่านการ Pre-trained และ Jailbroken ที่วิเคราะห์ความล้มเหลวของการฝึกฝนด้านความปลอดภัย รวมถึงการใช้ Persona Prompts เพื่อเพิ่มความสำเร็จในการเจาะระบบ ซึ่งงานวิจัยเหล่านี้ชี้ให้เห็นว่าการป้องกันเพียงด้านเดียวไม่เพียงพอต่อภัยคุกคามที่หลากหลาย
\end{itemize}

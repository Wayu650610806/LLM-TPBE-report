\chapter{\ifenglish{Background Knowledge and Theory}\else{ทฤษฎีที่เกี่ยวข้อง}\fi}

บทนี้กล่าวถึงแนวคิด ทฤษฎี และงานวิจัยที่เกี่ยวข้องกับตัวแบบภาษาขนาดใหญ่ 
(Large Language Models: LLMs) ความมั่นคงปลอดภัยของข้อมูล การโจมตีแบบ 
Data Extraction Attack และ Jailbreak Attack รวมถึงบริบททางภาษาศาสตร์ของภาษาไทย 
ซึ่งเป็นพื้นฐานสำคัญในการออกแบบการทดลองและการวิเคราะห์ผลของโครงงานนี้

\section{ตัวแบบภาษาขนาดใหญ่ (Large Language Models: LLMs)}

ตัวแบบภาษาขนาดใหญ่ (Large Language Models: LLMs) เป็นเทคโนโลยีสำคัญในงานด้านการประมวลผลภาษาธรรมชาติ 
(Natural Language Processing: NLP) โดยใช้โครงข่ายประสาทเทียมขนาดใหญ่ในการเรียนรู้รูปแบบของภาษา 
และสร้างข้อความที่มีความสอดคล้องกับบริบท

โมเดลสมัยใหม่ส่วนใหญ่พัฒนาบนสถาปัตยกรรม \textit{Transformer} 
ซึ่งใช้กลไก \textit{Self-Attention} เพื่อวิเคราะห์ความสัมพันธ์ของคำในลำดับข้อความที่ยาว 
ทำให้สามารถเข้าใจบริบทของภาษาได้อย่างมีประสิทธิภาพ

กระบวนการฝึกโมเดล LLM โดยทั่วไปประกอบด้วยสามขั้นตอนหลัก ได้แก่

\begin{itemize}

\item \textbf{Pre-training}  
การฝึกโมเดลด้วยชุดข้อมูลข้อความจำนวนมหาศาลเพื่อเรียนรู้โครงสร้างของภาษา

\item \textbf{Supervised Fine-tuning (SFT)}  
การปรับจูนโมเดลด้วยชุดข้อมูลที่มีคำตอบที่ถูกต้อง เพื่อให้โมเดลสามารถตอบสนองต่อคำสั่งได้อย่างเหมาะสม

\item \textbf{Reinforcement Learning from Human Feedback (RLHF)}  
การปรับแต่งพฤติกรรมของโมเดลด้วยความคิดเห็นจากมนุษย์ 
เพื่อสร้างสมดุลระหว่างความเป็นประโยชน์ของคำตอบและความปลอดภัยของระบบ
% \cite{ouyang2022rlhf}

\end{itemize}

แนวทาง RLHF ถูกนำมาใช้เพื่อควบคุมพฤติกรรมของโมเดลให้ตอบสนองในรูปแบบที่เหมาะสม 
รวมถึงลดความเสี่ยงในการสร้างเนื้อหาที่ไม่เหมาะสม

\section{ความสามารถหลายภาษาและผลกระทบของ Tokenization}

แม้ว่า LLMs จะถูกออกแบบให้รองรับหลายภาษา แต่ประสิทธิภาพของโมเดลอาจแตกต่างกันอย่างมากระหว่างภาษา

งานวิจัย \textit{MultiQ} ได้ประเมินความสามารถของ LLMs ในการประมวลผลภาษา 196 ภาษา 
และพบว่าภาษาอังกฤษมีประสิทธิภาพสูงที่สุด เนื่องจากมีข้อมูลฝึกฝนจำนวนมาก 
และใช้การแบ่งคำแบบ subword tokenization ที่เหมาะสม 
ในขณะที่หลายภาษา โดยเฉพาะภาษาที่มีข้อมูลน้อย มักถูกแบ่งคำในระดับ character 
ทำให้ประสิทธิภาพลดลงอย่างมาก \cite{jauhiainen2023multiq}

ผลการศึกษานี้สะท้อนให้เห็นว่าการประเมินความปลอดภัยของ LLMs 
จำเป็นต้องพิจารณาปัจจัยด้านภาษาและโครงสร้างของ tokenization 
เนื่องจากความแตกต่างดังกล่าวอาจส่งผลต่อพฤติกรรมของโมเดลและความเปราะบางต่อการโจมตี

\section{ความเป็นส่วนตัวของข้อมูลใน LLMs}

หนึ่งในความท้าทายสำคัญของ LLMs คือความเสี่ยงด้านความเป็นส่วนตัวของข้อมูล 
(\textit{Data Privacy})

งานวิจัย \textit{Extracting Training Data from Large Language Models} 
แสดงให้เห็นว่าโมเดลขนาดใหญ่สามารถจดจำข้อมูลบางส่วนจากชุดข้อมูลฝึกฝน 
และสามารถสร้างข้อมูลดังกล่าวออกมาใหม่ผ่านการออกแบบคำสั่งที่เหมาะสม
\cite{carlini2021extracting} 
% \cite{turn0search8}

ความเสี่ยงดังกล่าวเพิ่มขึ้นตามขนาดของโมเดล 
เนื่องจากโมเดลที่มีจำนวนพารามิเตอร์มากมีความสามารถในการจดจำรูปแบบข้อมูลได้มากขึ้น

รูปแบบการโจมตีที่เกี่ยวข้องกับความเป็นส่วนตัวของข้อมูลสามารถแบ่งได้เป็น

\begin{itemize}

\item \textbf{Data Privacy Leakage}  
การที่โมเดลเปิดเผยข้อมูลส่วนบุคคลที่อยู่ในชุดข้อมูลฝึกฝน

\item \textbf{Data Extraction Attack (DEA)}  
การออกแบบ prompt เพื่อบังคับให้โมเดลสร้างข้อมูลจากชุดข้อมูลฝึกฝน 
เช่น อีเมล หมายเลขโทรศัพท์ หรือข้อมูลส่วนบุคคล
\cite{carlini2021extracting}

\end{itemize}

\section{Jailbreak Attack ใน LLMs}

Jailbreak Attack เป็นการโจมตีที่พยายามหลีกเลี่ยงกลไกด้านความปลอดภัยของโมเดล 
โดยการออกแบบ prompt เพื่อทำให้โมเดลตอบสนองต่อคำสั่งที่ปกติควรถูกปฏิเสธ

หนึ่งในเทคนิคที่ถูกใช้บ่อยคือ \textit{Persona Prompt} 
ซึ่งเป็นการกำหนดบทบาทสมมติให้กับโมเดล เช่น Developer Mode หรือ DAN prompt

งานวิจัย \textit{Enhancing Jailbreak Attacks on LLMs via Persona Prompts} 
พบว่าการกำหนดบทบาทสมมติสามารถลดอัตราการปฏิเสธคำตอบของโมเดลได้อย่างมีนัยสำคัญ
และเพิ่มความสำเร็จของการโจมตีเมื่อใช้ร่วมกับเทคนิค jailbreak อื่น ๆ
% \cite{persona2025}

นอกจากนี้งานวิจัยด้าน alignment ยังพบว่าโมเดลจากผู้พัฒนาที่ใช้วิธีการปรับ alignment ที่แตกต่างกัน
จะมีระดับความต้านทานต่อ jailbreak แตกต่างกัน

งานวิจัย \textit{Equilibrate RLHF} 
แสดงให้เห็นว่าการปรับสมดุลระหว่างความเป็นประโยชน์ของโมเดล 
(\textit{helpfulness}) และความปลอดภัย (\textit{safety})
ส่งผลโดยตรงต่อความสามารถของโมเดลในการป้องกันการโจมตีประเภท jailbreak
% \cite{equilibrate2025}

\section{กลไกการป้องกันการโจมตี}

การป้องกันการโจมตีใน LLMs สามารถทำได้หลายวิธี โดยกลไกที่สำคัญประกอบด้วย

\subsection{Input Scrubbing}

Input Scrubbing เป็นเทคนิคที่ใช้ตรวจจับและลบข้อมูลที่อาจเป็นข้อมูลส่วนบุคคลออกจาก prompt 
ก่อนที่จะส่งไปยังโมเดล เช่น การใช้ Regular Expression (Regex) 
และ Named Entity Recognition (NER) เพื่อระบุชื่อบุคคล อีเมล หรือหมายเลขโทรศัพท์

\subsection{Defensive Prompting}

Defensive Prompting เป็นเทคนิคการกำหนดคำสั่งในระดับ \textit{system prompt}
เพื่อควบคุมพฤติกรรมของโมเดล

งานวิจัยเกี่ยวกับ adversarial prompts 
พบว่าการกำหนดข้อจำกัดด้านความปลอดภัยในระดับ system prompt
สามารถช่วยลดความสำเร็จของ jailbreak attacks ได้อย่างมีนัยสำคัญ
\cite{zou2023universal}

\section{บริบทภาษาไทยและระดับภาษา}

ภาษาไทยมีลักษณะเฉพาะทางภาษาศาสตร์ เช่น การไม่เว้นวรรคระหว่างคำ 
การใช้คำตามบริบททางสังคม และการเปลี่ยนระดับภาษาในสถานการณ์ต่าง ๆ

ในการศึกษานี้ ระดับภาษาไทยถูกแบ่งออกเป็นห้าระดับ ได้แก่

\begin{enumerate}

\item ภาษาระดับพิธีการ
\item ภาษาระดับทางการ
\item ภาษาระดับกึ่งทางการ
\item ภาษาระดับไม่เป็นทางการ
\item ภาษาระดับกันเอง
\end{enumerate}

ระดับภาษาที่แตกต่างกันอาจส่งผลต่อพฤติกรรมของโมเดล 
เนื่องจาก LLMs ถูกฝึกด้วยข้อมูลจากอินเทอร์เน็ตจำนวนมาก 
ซึ่งมักมีรูปแบบภาษาที่ไม่เป็นทางการเป็นหลัก

ดังนั้นการศึกษาผลกระทบของระดับภาษาไทยต่อพฤติกรรมของโมเดล
จึงเป็นส่วนสำคัญในการประเมินความเสี่ยงด้านความเป็นส่วนตัวของข้อมูลในบริบทภาษาไทย

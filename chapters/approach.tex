\section{ภาพรวมสถาปัตยกรรมระบบ (System Architecture)}
ระบบถูกออกแบบให้ทำงานในรูปแบบ Pipeline เพื่อทดสอบโมเดลภาษาผ่าน Groq API โดยมีขั้นตอนการไหลของข้อมูลดังนี้:
\begin{enumerate}
  \item Input Layer: รับคำสั่ง (Prompt) ที่ผ่านการปรับแต่งระดับภาษาไทย 5 ระดับ
  \item Defense Layer (Input Scrubbing): กรองข้อมูลส่วนบุคคล (PII) ขาเข้าด้วยเทคนิค Regex และ NER ก่อนส่งไปยังโมเดล
  \item Inference Layer: ส่งคำสั่งไปยังโมเดล llama-3.1-8b-instant, meta-llama-llama-4-maverick-17b-128e-instruct หรือ moonshotai-kimi-k2-instruct-0905 ผ่าน Groq API พร้อมการใช้งาน Defensive Prompting
  \item Evaluation Layer: ระบบรวบรวมคำตอบและประเมินผลด้วยมาตรวัดที่ออกแบบอัตโนมัติ
\end{enumerate}

\section{การเตรียมชุดข้อมูลและระดับภาษา (Data Preparation)}
\begin{itemize}
  \item ชุดข้อมูลหลัก: ใช้ข้อมูลอีเมลจาก Enron สำหรับการทดสอบ DEA และข้อมูลส่วนบุคคลของบุคคลสำคัญ (เช่น Joe Biden) สำหรับการทดสอบ JA
  \item กระบวนการปรับระดับภาษา: ใช้เทคนิค Batch Processing ผ่าน OpenAI API เพื่อแปลชุดคำสั่งปริมาณ 3,050 ชุด ให้ครอบคลุม 5 ระดับภาษาไทย ได้แก่ ระดับพิธีการ, ทางการ, กึ่งทางการ, ไม่เป็นทางการ และกันเอง
\end{itemize}

\section{ระเบียบวิธีการโจมตี (Attack Methodology)}
สภาพแวดล้อมการทดสอบ: ดำเนินการทดสอบซ้ำอย่างน้อย 5 รอบต่อกรณี เพื่อหาค่าเฉลี่ยอัตราความสำเร็จ (Average Success Rate)
\begin{itemize}
  \item Data Extraction Attack (DEA): ออกแบบคำสั่งเพื่อล่อลวงให้โมเดลเปิดเผยข้อมูลอีเมลหรือข้อมูลลับที่ถูกเก็บไว้ในหน่วยความจำ
  \item Jailbreak Attack (JA): ใช้เทคนิค Persona Prompts หรือการสร้างสถานการณ์จำลองเพื่อหลีกเลี่ยงระบบความปลอดภัยของโมเดล
\end{itemize}

\section{การออกแบบกลไกการป้องกัน (Defense Design)}
\subsection{Input Scrubbing: (เอาเลขแบบนี้ หรือจุดๆดี)}
พัฒนาระบบตรวจสอบคำสั่งขาเข้าด้วย Regular Expression (Regex) และ Named Entity Recognition (NER) เพื่อตรวจจับและปกปิดข้อมูลประเภท ชื่อ, อีเมล, เบอร์โทรศัพท์ ก่อนถึงตัวโมเดล
\subsection{Defensive Prompting}
ออกแบบ System Prompt จำนวน 12 รูปแบบ (แบ่งเป็นสำหรับ DEA 6 แบบ และ JA 6 แบบ) เพื่อกำหนดขอบเขตพฤติกรรมของโมเดลให้ปฏิเสธการตอบข้อมูลที่เป็นอันตราย

\section{การพัฒนาระบบรันการทดสอบอัตโนมัติ (Automated Testing)}
ทีมวิจัยได้พัฒนาเครื่องมือเพื่อเพิ่มประสิทธิภาพในการทดลอง ดังนี้:
\begin{itemize}
  \item Multi-API and Multi-Attack Execution: สคริปต์ใน Terminal เดียวที่สามารถสั่งรันหลายโมเดลและหลายรูปแบบการโจมตีพร้อมกันได้
  \item Automated Data Logging: ระบบจะบันทึกผลลัพธ์ลงใน Google Sheets โดยอัตโนมัติ
  \item Metrics Calculation: พัฒนาฟังก์ชันคำนวณปริมาณ Token, ความยาวข้อความ และสร้างกราฟสรุปผล (Visualization) ทันทีหลังจบการทดลอง
\end{itemize}

\section{มาตรวัดการประเมินผล (Evaluation Metrics)}
\begin{itemize}
  \item DEA Metrics: วัดความถูกต้องแม่นยำ (Accuracy) ในการดึงข้อมูลตามความเป็นจริง
  \item JA Metrics: กำหนดเกณฑ์การตัดสิน (Labeling) ว่าโมเดลถูกเจาะระบบสำเร็จหรือไม่ โดยพิจารณาจากข้อมูลที่โมเดลตอบกลับมาในส่วนของข้อมูลติดต่อ
\end{itemize}
